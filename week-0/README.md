# Nädal 0: Onboarding

## Kontekst
UrbanStyle.ltd vajab professionaalset andmesüsteemi — 
andmebaasi, versioonihaldust ja ühist töökeskkonda. 
Selle nädala eesmärk oli seadistada kõik tööriistad 
ja luua meeskonnana ühine keskkond.

## Kasutatud tööriistad
Seadistatud ja aktiivselt kasutatud:
- **Supabase** — andmebaas pilves
- **GitHub** — versioonihaldus ja portfoolio
- **Git** — lokaalne versioonihaldus
- **VS Code** — koodiredaktor (ühendatud Supabase'iga)
- **NotebookLM** — AI õppekaaslane RAG failidega
- **Claude** ja **ChatGPT** — AI abistavad tööriistad

Alla laaditud, kuid veel aktiivselt mitte kasutatud:
- **Python** — installitud ja ühendatud
- **Power BI** — visualiseerimistööriist (Track A)

## Isiklik töö
Seadistasin kõik töökeskkond ja ühendasin need 
omavahel — VS Code, Supabase ja GitHub töötavad 
nüüd koos ühe töövooks.

## Grupitöö — Roll D: Team Charter Koostaja
**Meeskond:** Operations Intelligence

Kogusin meeskonnalt info ja täitsin Team Charteri:
- leppisime kokku suhtluskanali (Google Workspace Chat)
- planeerisime rollide rotatsiooni 4 nädalaks
- sisestasin charter andmed Supabase `team_charter` tabelisse
- commit'isin `charter.md` faili meeskonna GitHub reposse

## Peamised leiud
- Andmebaas lahendab Exceli põhiprobleemid: 
  üks versioon, mitu kasutajat korraga
- Commit on nagu "save point" — töö ei kao kunagi
- Kõik tööriistad töötavad koos: 
  VS Code → Supabase → GitHub
- **NotebookLM** oli nädala suurim üllatus — 
  RAG failidel põhinev AI õppekaaslane, mis aitab 
  materjali läbi töötada, küsimustele vastata 
  ja audio ülevaateid genereerida

## Järgmised sammud
Nädal 1-s hakkame päris SQL päringutega 
UrbanStyle'i müügiandmeid uurima.
